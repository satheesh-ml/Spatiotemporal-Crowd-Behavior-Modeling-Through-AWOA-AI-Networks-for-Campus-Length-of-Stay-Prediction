# Spatiotemporal-Crowd-Behavior-Modeling-Through-AWOA-AI-Networks-for-Campus-Length-of-Stay-Prediction

4. Proposed Methodology: CLT-AWO Framework
4.1 System Overview
The proposed CLT-AWO framework comprises four sequential stages: (1) Data Preprocessing and Feature Engineering, (2) Feature Encoding and Representation, (3) CNN-LSTM-Transformer (CLT) Ensemble Modeling, and (4) Adaptive Whale Optimization Algorithm (AWOA) for joint hyperparameter tuning. 4.2 Data Preprocessing and Feature Engineering
Raw check-in and check-out timestamps (stored as Excel fractional day values) are decoded into hours and minutes using the formula: time_hours = fractional_value × 24. LOS is computed as: LOS = check-out_hours − check-in_hours, handling midnight-crossing cases. Date values are decoded from Excel serial numbers to obtain day-of-week, week-of-semester, and semester-week index. Outlier LOS values (< 0 or > 10 hours) are removed (0.8% of records). Missing demographic values are imputed using mode for categorical variables and median for numeric variables. For the Foursquare dataset, LOS is estimated as the inter-visit time gap, and records exceeding 8 hours are excluded as likely data gaps rather than actual visits.
Feature engineering generates 14 derived features from raw data:
1.	Temporal features: hour_of_day, minute_of_hour, day_of_week, is_weekend, time_slot (morning/afternoon/evening/night)
2.	Spatial features: GPS cluster label (k-means, k=7), distance_to_centroid, POIL_encoded
3.	Demographic features: gender_encoded, year_of_study, age_normalized, program_encoded
4.	Sequential features: previous_POIL, visit_count_today, mean_LOS_per_student (rolling)

4.3 Feature Encoding
POIL categorical labels are encoded using three complementary strategies applied to different model branches. Label Encoding maps each POIL string to an integer index (0–6), suitable for the CNN input grid construction. One-Hot Encoding produces binary indicator vectors, used as explicit categorical inputs to the LSTM branch to avoid imposing ordinal assumptions. Target Encoding replaces each POIL category with the mean LOS of that category computed on the training split only (to prevent data leakage), used in the Transformer branch for a richer semantic representation. Geographic state labels are likewise target-encoded using mean LOS per state. Figure 2 illustrates the three encoding schemes applied to POIL data.

4.4 CNN Sub-Model
The CNN branch receives a 2-D feature matrix constructed by arranging the 14 engineered features into a 7×2 grid after min-max normalization. This spatial arrangement groups temporally related features (columns 0-3) and spatially related features (columns 4-6) into contiguous regions, enabling convolutional kernels to detect feature co-occurrence patterns. The architecture consists of three convolutional blocks, each comprising a Conv2D layer (kernel size 3×3, padding same), Batch Normalization, ReLU activation, and MaxPooling2D (pool size 2×2). Filter counts are 64, 128, and 256 for the three blocks, respectively. A Global Average Pooling layer flattens the spatial maps to a 256-dimensional feature vector fed to a fully connected layer (128 units, ReLU).
4.5 LSTM Sub-Model
The LSTM branch receives variable-length visit sequences for each student within a rolling 5-visit window, encoded as a 5×d feature matrix where d is the one-hot encoded feature dimension. A two-layer stacked LSTM (units: [128, 64]) with dropout (0.3) models sequential visit patterns. Bidirectional wrapping of the first LSTM layer enables the network to exploit both past and look-ahead context within the sequence window. The final hidden state of the second LSTM layer produces a 64-dimensional vector.
4.6 Transformer Sub-Model
The Transformer branch receives the full feature vector (target-encoded POIL + temporal + demographic features, dimension 20) treated as a sequence of length T=10 (time-step chunks of a student's daily trajectory). A positional encoding layer injects temporal order information. The encoder stack comprises 4 multi-head self-attention layers (8 heads, key dimension 64) interleaved with position-wise feed-forward networks (FFN dimension 256) and layer normalization. Global average pooling over the sequence dimension yields a 256-dimensional context vector.
4.7 Ensemble Fusion
The three branch outputs (CNN: 128-dim, LSTM: 64-dim, Transformer: 256-dim) are concatenated to form a 448-dimensional joint representation. Two dense layers (256 units → 128 units, both ReLU, dropout 0.4) reduce this to a final scalar LOS prediction through a single linear output neuron. Branch-level attention weighting is applied before concatenation: a softmax-normalized weight vector (α_CNN, α_LSTM, α_Transformer) learned via a small attention network emphasizes the most informative branch per input sample.
4.8 Adaptive Whale Optimization Algorithm (AWOA)
Standard WOA updates whale positions (hyperparameter vectors) according to the shrinking encircling mechanism and the spiral bubble-net attack pattern. Our AWOA introduces two enhancements:
Enhancement 1 — Nonlinear Adaptive Inertia Weight (NAIW): The inertia coefficient w(t) decays nonlinearly according to: w(t) = w_max × exp(−λ × (t/T_max)²), where w_max = 0.9, λ = 2.0, t is current iteration, and T_max is maximum iterations. This ensures aggressive exploration early in search and refined exploitation later, avoiding the premature convergence observed in standard WOA.
Enhancement 2 — Levy-Flight Perturbation (LFP): At each iteration with probability p_levy = 0.2, a whale's position is perturbed by a Levy-distributed step: X_i(t+1) = X_i(t) + σ × Lévy(β), where β = 1.5 and σ is a scale parameter proportional to the current search space extent. Levy flights have heavy tails that enable occasional large jumps, escaping local optima more effectively than Gaussian perturbations.
AWOA optimizes a 22-dimensional hyperparameter vector spanning CNN filter counts (3), LSTM units (2), Transformer heads and FFN dimension (2), dropout rates (3), learning rate (1), batch size (1), and ensemble attention temperature (1), plus 9 additional architectural choices. The fitness function is the 5-fold cross-validation MSE on the training set, evaluated over 50 AWOA iterations with a population of 30 whales. 4.9 Training Protocol
All experiments are implemented in Python 3.10 using TensorFlow 2.13 and Keras. Datasets are split 70:15:15 into training, validation, and test sets with stratified sampling by POIL category. Adam optimizer is used with the AWOA-tuned learning rate. Early stopping (patience=10) on validation MSE prevents overfitting. All results are averaged over 5 independent runs with different random seeds to ensure robustness. The Foursquare model is trained on the NYC subset and zero-shot tested on a held-out DSOLP subset to evaluate cross-domain transferability.
